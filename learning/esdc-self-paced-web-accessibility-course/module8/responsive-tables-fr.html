Conception de tableaux adaptés

Les tableaux adaptés sont redimensionnés à mesure que la taille de la zone d’affichage change pour s’assurer qu’ils ne nécessitent pas de défilement horizontal. Pour les tableaux comportant quelques colonnes, on peut le faire en donnant à chaque colonne un pourcentage de largeur et en donnant au tableau une largeur maximale de 100 %. Les tableaux plus grands doivent être transformés et réorganisés en une seule colonne. Veuillez noter que le gouvernement du Canada n’a pas besoin de tableaux adaptés : ni le système de conception de Canada.ca ni la BOEW ne comportent de fonctionnalité permettant de transformer et de réorganiser les tableaux, comme le montre l’exemple ci-dessous.

Si l’écran est trop petit pour afficher le tableau complet, une barre de défilement horizontale s’affiche.

 Bon exemple : Reformater le tableau en une seule colonne

Dans l’exemple ci-dessous, le tableau est reformaté à une colonne si la largeur du navigateur est inférieure à 768 pixels, ce qui maintient la lisibilité sans qu’un défilement horizontal soit nécessaire.

La technique consiste à empiler les cellules du tableau les unes sur les autres, à masquer les en-têtes au haut de la colonne et à les reproduire à gauche de la cellule comme des en-têtes de rangée visuels. Le problème avec cette technique est que les en-têtes de rangée sont créés avec le contenu généré par CSS, en s’appuyant sur la valeur d’un attribut personnalisé d’étiquette de données:

Début du code

<td data-label="Account">Mastercard - 5555</td>
@media screen and (max-width: 767px) {
   table td::before {
      content: attr(data-label);
      float: left;
      font-weight: bold;
      text-transform: uppercase;
   }
}

Fin du code

Malheureusement, le contenu généré par CSS n’est pas encore déclaré par les lecteurs d’écran JAWS et Narrator. Une fois qu’ils pourront le faire, cette technique sera viable.

Début du code

table { 
   border: 1px solid #ccc; 
   border-collapse: collapse; 
   margin: 0; 
   padding: 0;
   width: 100%;
   table-layout: fixed; 
} 
table caption { 
   font-size: 1.2em; 
   margin: .5em 0 .75em;
} 
table th, 
table td {
   padding: .625em; 
   text-align: center; 
   border: 1px solid #CCC;
} 
table thead th {
   letter-spacing: .1em;
   text-transform: uppercase; 
} 
@media screen and (max-width: 767px) { 
   table { 
      border: 0; 
   } 
   table caption { 
      font-size: 1.2em; 
   } 
   table thead { 
      border: none;
      clip: rect(0 0 0 0); 
      height: 1px; 
      margin: -1px; 
      overflow: hidden; 
      padding: 0;
      position: absolute; 
      width: 1px;
   } 
   table tr { 
      border-bottom: 3px solid #ddd;
      display: block;
      margin-bottom: 1em;
   }
   table td, table tbody th { 
      border-bottom: 1px solid #ddd; 
      display: block; 
      text-align: right; 
      width:95%;
   } 
   table td::before, 
   table tbody th::before { 
      content: attr(data-label); 
      float: left; 
      font-weight: bold; 
      text-transform: uppercase; 
   }
   table td:last-child { 
      border-bottom: 0; 
   } 
}

Fin du code

 Bon exemple : Débordement du tableau avec barre de défilement horizontale.

Dans cet exemple, un grand tableau force une barre de défilement horizontale sur l’élément <div> parent, générée par la déclaration CSS overflow-x: auto. Le div utilise également l’attribut tabindex="0" pour s’assurer que la barre de défilement reçoit la cible du clavier (conformément à la règle W3C-IAW TIA Scrollable element is keyboard accessible (l’élément de défilement est accessible au clavier)).

Il est toujours préférable de diviser les grands tableaux en plusieurs petits tableaux.

Début du code

<div style="overflow-x:auto;" tabindex="0">
   <table>[…]</table>
</div>
