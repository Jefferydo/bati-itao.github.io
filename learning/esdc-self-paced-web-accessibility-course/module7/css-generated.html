Designing with CSS-generated content

CSS can generate text and icons through the :before and :after pseudo-classes. However, CSS-generated content is not in the DOM and consequently not easily accessible to assistive technology. While the NVDA and VoiceOver screen readers recognize CSS-generated content, the JAWS and Narrator screen readers don’t.

Consequently, meaningful CSS-generated content can be used on print style sheets to print the URL of links, but otherwise it should be avoided.

If you use CSS-generated content, hide it from screen readers with the aria-hidden="true" attribute. If the CSS-generated content is informative, provide a text alternative. To provide a text alternative:

  • If the content is a link or control, use the aria-label attribute.
  • If the content can't receive focus, use the WET .wb-inv class to visually hide the text alternative.

 Good example: Informative CSS icon

In this example, the Font Awesome hammer icon is used to represent a task.

  • The <span> element with classes .fas and .fa-hammer displays the icon via a CSS :before pseudo-class.
  • Its title attribute holds a tooltip for mouse users, "Task".
  • Its aria-hidden="true" attribute hides the element from screen readers, since their support for the title attribute is inconsistent.
  • Directly following the icon is the visually-hidden name “Task”, which declares in all screen readers.

Task Complete the application

HTML

Code begins

<p>
   <span class="fas fa-hammer" title="Task" aria-hidden="true"></span>
   <span class="wb-inv">Task</span>
   Complete the application
 </p>

Code ends

CSS

Code begins

.fa-hammer:before { content: "\f6e3"; }
.fas { font-family: "Font Awesome 5 Free"; }
.wb-inv {
   clip: rect(1px,1px,1px,1px);
   height: 1px;
   margin: 0;
   overflow: hidden;
   position: absolute;
   width: 1px;
}

Code ends
