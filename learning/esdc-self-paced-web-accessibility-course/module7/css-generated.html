Designing with CSS-generated content

CSS can generate text through the :before and :after pseudo-classes, but the text is not in the DOM and consequently not easily accessible to assistive technology. While the NVDA and VoiceOver screen readers recognize CSS-generated content, the JAWS and Narrator screen readers don’t.

Consequently, meaningful CSS-generated content can be used on print style sheets to print the URL of links on the page, but otherwise it should be avoided.

If you do use CSS-generated content, hide it from screen readers with the aria-hidden=”true” attribute. If the CSS-generated content is informative, provide a text alternative. To provide a text alternative:

  • If the content is a link or control, use the aria-label attribute.
  • If the content can't receive focus, use the WET .wb-inv class to visually hide the text alternative.

Example: Visually-hiding CSS-generated content and providing a text alternative

In this example, the Font Awesome hammer icon is used to represent a task. The span element with class “fas fa-hammer" displays the icon via a CSS pseudo-class, while its title attribute holds a tooltip for mouse users and the aria-hidden=”true” attribute hides the element from screen readers. Directly following is the visually-hidden name “Task”, which declares in screen readers.
