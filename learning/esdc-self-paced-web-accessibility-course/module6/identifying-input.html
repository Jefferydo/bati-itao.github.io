Identifying input purpose (overview)

When a form input field collects personal information such as the user’s name, phone number, email and street addresses, you must programmatically define its semantics with the autocomplete attribute. The attribute takes one of 53 potential values from WCAG’s list of Input Purposes for User Interface Components. The values are granular; for instance, regarding the user’s name, it’s possible to identify an input’s purpose as full name, given name, family name, or nickname.

Assistive technologies leverage the autocomplete attribute value to automatically complete forms, drawing on past user input stored in the browser. This saves the user from having to type the information. Assistive technologies also leverage the autocomplete value to pair input fields with icons, for users who prefer using images for communication.
