Identifying input purpose (overview)

FRENCH: re-arranged sentences below

When a form input field collects personal information such as the user’s name, phone number, email and street addresses, you must programmatically define its semantics with the autocomplete attribute. Assistive technologies leverage the autocomplete attribute value to automatically complete forms, drawing on past user input stored in the browser. This saves the user from having to type the information. Assistive technologies also leverage the autocomplete value to pair input fields with icons, for users who prefer using images for communication.

The attribute value takes one of 53 potential values field names from WCAG’s list of Input Purposes for User Interface Components. The values are granual The field names are either broad or narrow. For instance, regarding the user’s name, it’s possible to identify an input’s purpose as full name, given name, family name, or nickname. You can further refine the purpose by adding space-separated autofill detail tokens to the autocomplete attribute value:

  • shipping
  • billing
  • home
  • work
  • mobile
  • fax
  • pager

Use autocomplete on <input> elements that take a text or numeric value as input, <textarea> elements, <select> elements, and <form> elements. The HTML5 specification lists the appropriate elements for the different autocomplete values.

<form method="post" action="step1">  
   <label for="fname">First name</label>
   <input id="fname" type="text" autocomplete="given-name">  

   <label for="lname">Last name</label>  
   <input id="lname" type="text" autocomplete="family-name">    
   
   <label for="jobtitle">Job title</label>  
   <input type="text" id="jobtitle" autocomplete="organization-title">
   
   <label for="org">Organization</label>
   <input type="text" id="org" autocomplete="organization">
   
   <fieldset>
      <legend>Business address</legend>
      
      <label for="work-st">Street</label>
      <input type="text" id="work-st" autocomplete="work street-address">
      
      <label for="work-city">City</label>
      <input type="text" id="work-city" autocomplete="work address-level2">
      
      <label for="work-prov">Province</label>
      <input type="text" id="work-prov" autocomplete="work address-level1">
      
      <label for="work-mobile-phone">Mobile phone for delivery</label>
      <input type="tel" id="work-mobile-phone" autocomplete="work mobile tel-national">
      
      <label for="work-email">Email</label>
      <input type="email" id="work-email" autocomplete="work email">
   </fieldset>
   
   <fieldset>
      <legend>Home address</legend>
      
      <label for="home-st">Street</label>
      <input type="text" id="home-st" autocomplete="home street-address">
      
      <label for="home-city">City</label>
      <input type="text" id="home-city" autocomplete="home address-level2">
      
      <label for="home-prov">Province</label>
      <input type="text" id="home-prov" autocomplete="work address-level1">
      
      <label for="home-phone">Phone</label>
      <input type="tel" id="home-phone" autocomplete="home tel-national">
      
      <label for="home-email">Email</label>
      <input type="email" id="home-email" autocomplete="home email">
   </fieldset>
     
   <input type="submit" value="Continue"> 
</form>
